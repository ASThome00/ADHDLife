## PART 1 — CODE REVIEW

### Critical

**C1. Snoozed tasks never come back — they vanish forever.**
`useSnoozeTask` sets `status='SNOOZED', snoozed_until=...`, but no code anywhere reads `snoozed_until` to un-snooze. Every view filters SNOOZED out (dashboard overdue, tasks page, review carried-over). "Snooze 1 day" is functionally "delete with a 3-second undo window." Worst violation of Design Law #6 (object permanence).
Fix: on app/dashboard load run `UPDATE tasks SET status='ACTIVE', snoozed_until=NULL WHERE status='SNOOZED' AND snoozed_until <= ?`.

**C2. Recurrence UI is a no-op — nothing ever spawns recurring instances.**
Task detail panel offers Repeat: Daily/Weekly/… and writes the `recurrences` row, but `last_spawn_at` is never read or written anywhere. "Take meds — Daily" completed once never appears again. Silent feature failure burns trust.
Fix: spawn-on-launch routine (compare `last_spawn_at`/`completed_at` to frequency, clone task), or hide the Repeat control until it works.

**C3. Habit dates are UTC-keyed — evening check-ins land on tomorrow.**
`todayDateStr()` = `toISOString().split('T')[0]` (UTC). From ~7–8pm US-local onward this is tomorrow's date: checking a habit at 9pm writes tomorrow's log row; next morning the circle already shows "done"; the 30-day grid shows tonight's check-in on tomorrow's cell; the streak walk walks UTC dates; review `windowDates` can count "future" logs (habit % can exceed 100).
Fix: one local-date helper (`format(new Date(), 'yyyy-MM-dd')`) used by `todayDateStr`, `getHabitHistory`, the dot grid, the streak walk, and `useToggleHabit`'s optimistic key.

**C4. Tasks-page "Today" grouping disagrees with due-date storage.**
Due dates are stored as local-23:59 → UTC ISO (next UTC day for US timezones). The Tasks page groups by `due_date.slice(0,10)` (UTC date). A "Today" quick-add shows label "Today" but sits in **This week**; "Move to today" files under tomorrow. Dashboard compares instants so it's unaffected — making Tasks look randomly broken by contrast.
Fix: group by local date (`new Date(t.due_date)` → local y-m-d), never `slice(0,10)`.

### Bugs

- **B1. `ORDER BY priority DESC` sorts TEXT alphabetically — HIGH sorts LAST** (`getTasks`, dashboard `upcomingToday`, copied in mobile db.ts). Fix with a CASE map.
- **B2. Checking a subtask does nothing visible** — `useCompleteTask` never invalidates `qk.subtasks`; checkbox stays unchecked until remount.
- **B3. Task detail panel shows stale values after every edit** — panel holds a frozen Task snapshot; priority/category pills never light up after clicking. Store the id, derive the task from live cache.
- **B4. Habit streaks only recalc on toggle** — miss 3 days, card still says 🔥10; next tap drops it to 1 unexplained. Also un-checking today's circle zeroes the streak even when yesterday's chain is intact. Compute streak on read (ending today or yesterday).
- **B5. Custom category colors don't work; two color systems fight** — new categories render fallback brown everywhere that uses `getCategoryTheme` (keyed to 8 seeded ids); `CategorySidebar` + detail-panel picker read raw DB colors (bright legacy hexes); sidebar's new-category swatches are the legacy palette, not `PRESET_COLORS`; Settings recolor writes a column almost nothing reads.
- **B6. DB failure at startup silently shows the Setup screen** (settings query error → `undefined` → first-run branch). Terrifying and can overwrite her name. No page has an error state at all.
- **B7. Undo toast + Inbox bulk bar render off-center** — framer-motion `y` animation overrides `translateX(-50%)`. Animate `x: '-50%'` too.
- **B8. `daily_focus_limit` is a placebo** — dashboard hardcodes LIMIT 5; nothing caps focus adds; `is_focus_today` never resets at day rollover, so focus accumulates and overflow (>5) becomes invisible.
- **B9. Weekly-review optimistic removals have no rollback; no mutation invalidates `qk.weeklyReview`** — stats stale after actions.
- **B10. Category sidebar counts inflated** — includes SNOOZED tasks and subtasks.

### Paper-cuts

- P1. Inbox rows hide locally before mutate; on error the task stays hidden until remount.
- P2. Settings inputs get clobbered by refetches (`useEffect` on the whole settings object resets in-progress typing when any other setting mutates).
- P3. Focus card ratio counts ALL completions today, not focus completions — "3/3" with zero focus tasks.
- P4. Dashboard caps at 6 habit circles with no "+N more" hint.
- P5. Archived habits unrecoverable in UI (no undo, no archived list).
- P6. Fonts load from Google Fonts at runtime — offline = identity gone; per-launch ping in a "local-only" app. Bundle locally.
- P7. Setup + loading screens use the abandoned violet/stone palette (`bg-surface-50`, `text-primary-400`, `ring-primary-400`) — first-run and cold-start flash a different app.
- P8. `toggleHabitToday` streak walk is one SELECT per day through Tauri IPC (200-day streak = 200 round trips per tap) and non-transactional.
- P9. `brainDump` inserts serially without a transaction — mid-loop failure leaves a partial dump.
- P10. Review habit % lacks an upper date bound (interacts with C3; can exceed 100%).
- P11. Quick-add Escape discards typed text, no draft; "This week" secretly means "next Friday" (6 days out on a Saturday).
- P12. Overdue section excludes overdue INBOX tasks (they show only in Inbox with no overdue cue); INBOX tasks with due dates appear in BOTH "Today" and "Inbox" sections (duplicate rows).
- P13. `fs:allow-write-text-file` scoped to `**` — narrow to `$HOME/**`/`$DOWNLOAD/**`.
- P14. Mobile `lib/db.ts` schema drift: missing `recurrences`/`tags`/`task_tags`/`focus_days`; settings lacks columns AND never seeds row '1'; theme default differs; category ids `cat_0…cat_7` vs desktop `cat_work…` (breaks future sync + shared CATEGORY_THEME); priority-sort bug copied.

### Nits

- N1. Four priority-color definitions (`PRIO`, `MODAL_PRIO_COLORS`, `PRIO_COLORS`, dead `PRIORITY_CONFIG`) — dots don't match picker pills. Consolidate.
- N2. Dead code: `isOverdue()`, `packages/types` unused by desktop, `tags`/`focus_days` tables codeless, notification plugin registered with no notification code.
- N3. `updateTask` can emit duplicate `status =` assignments (status + category_id together) — latent trap.
- N4. `updateTask` nulls `completed_at` on any non-DONE status write — snoozing a completed task erases completion history.
- N5. "Paused at X days" uses `longest_streak` — "Paused at 34" after a 3-day streak reads oddly.
- N6. A11y: kebab menus lack aria/Escape; detail panel lacks Escape/focus trap/role="dialog"; SubCheckbox not focusable; task rows not keyboard-reachable; UndoToast lacks role="status".
- N7. Section collapse animates from maxHeight 99999 — looks instant/janky.
- N8. `PRAGMA journal_mode = WAL` inside migration 001 likely doesn't take effect (transactional migration; per-connection pragma). Verify WAL at runtime if it matters.
- N9. `getDueDate` duplicated in quick-add-modal and inbox.
- N10. Tauri/updater config is sane (endpoints, pubkey, manual check, migrations registered).

## PART 2 — PRODUCT / UX AUDIT

### The three headline betrayals (all Law #6 violations)

1. **Snooze is a black hole** (= C1).
2. **Recurrence never spawns** (= C2).
3. **There is no way to make an existing task a focus task.** FocusTasksCard says "pick a few from your task list", but the tasks kebab has only Edit/Snooze/Move-to-today/Drop and the detail panel has no focus toggle. The only paths to focus are quick-adding a NEW task from the dashboard or "Focus now" buried in Weekly Review. The hero screen's core loop is unclosable.

### Dashboard
- Gaps: 6am empty state is a dead end (no picker exists); focus fraction wrong (counts all completions); completed focus tasks vanish instead of striking through (visible progress = dopamine); `is_focus_today` never resets (unused `focus_days` table was clearly meant for this); WeekStrip is decorative — answers no question; habit circles cap at 6 silently; no "what did I do today" anywhere.
- Quick wins: "Focus today" action on tasks (~1h, repairs the core loop); keep DONE focus tasks visible struck-through + fix the math; empty focus card → 3–5 one-tap suggestions from due/overdue already in the same query; wire `daily_focus_limit`; expand MotivationQuote to ~30 seeded-by-date; show review's next-week priorities Mon–Wed; WeekStrip dots with due counts.
- Bigger: **"Plan my day" morning ritual** — first open of the day: carry over yesterday's unfinished focus? pick ≤N focus, glance habits; writes a `focus_days` row, resets stale flags. Must be skippable in one click. **Time-of-day awareness** — evening open shows "today you finished X, Y, Z" + tomorrow preview.
- Questions: auto-carry unfinished focus vs ask each morning? Is the "X of Y done" topbar pill motivating or a scoreboard?

### Inbox
- Gaps: bulk "Set due date" leaves INBOX status (causes the duplicate-row bug); drop has no undo here (inconsistent); no global capture — QuickAdd is a decision UI, not a capture UI; no aging cues.
- Quick wins: global in-app Ctrl+N straight to dump-mode quick-add; undo toast for inbox drop; promote-to-ACTIVE on bulk due date; faint age hint ("3w"); quiet dashboard line "Inbox: 7 unsorted".
- Bigger: **OS-level global capture** (Tauri global shortcut, minimal always-on-top dump window even when app closed) — arguably the highest-leverage ADHD feature Tauri uniquely enables. **Triage mode** — one-card-at-a-time inbox processing with keyboard (1–8 category, D drop, S snooze).
- Question: does she sort the inbox or does it accumulate? (Determines triage vs capture investment.)

### Tasks
- Gaps: no focus action (headline); snoozed tasks invisible AND never wake; dropped tasks unrecoverable after 3s (= delete with soft branding); priority sort broken; detail panel missing due-time/estimate/focus/snooze/drop, can't uncheck or delete subtasks, weekly recurrence can't pick days; no done visibility; no search.
- Quick wins: priority CASE sort (15 min); "Focus today ⚡" in kebab + panel; snooze wake-up (2 lines) + collapsed "Snoozed (3)" section; collapsed "Recently done" + "Dropped → pick back up" sections; surface `time_estimate_min` as pills (5m/15m/30m/1h) + faint display on rows; render seeded category icon emojis.
- Bigger: **recurrence spawner** (a day of date math; until then hide the control); **time-estimate feedback loop** (estimated vs actual, aggregate-only, never per-task shame — time blindness is THE deficit); **Cmd+K search across all statuses** ("I know I wrote it down" is object-permanence rescue).
- Questions: should Someday items resurface proactively (e.g. one random per weekly review)? Is a Dropped archive comforting or a guilt pile?

### Habits
- Gaps: the UTC bug (= C3); no way to log yesterday ("did it, forgot to tap") — grid is display-only; `target_frequency` unused so a 3×/week gym habit reads as perpetual failure; archive has no unarchive; "paused" copy contradicts the 🔥0 number.
- Quick wins: local-date fix; make last ~3 grid dots tappable to backfill; show "⏸ 12d" instead of 🔥0 when paused; collapsed archived list with Restore.
- Bigger: **weekly-target habits** ("3 of 5 this week" ring; streak = weeks target met) — removes the biggest source of fake failure; streak sparkline popover from dashboard circles.
- Questions: are her habits daily, weekly, or mixed? Does she want streak numbers at all, or are counters themselves the anxiety?

### Reading
- Gaps: one-way flow (no un-finish, no move-back, no remove); `ABANDONED` exists in schema with no UI — "it's okay to put a book down" is the most ADHD-compassionate feature this page could have; no reading↔task bridge though med-school reading is her real workload.
- Quick wins: card ⋯ menu (Move back / "Put down for now" → ABANDONED shown as a soft "Resting" row / Remove); "+10 pages" one-tap button; faint "started X days ago".
- Bigger: "Read tonight" button creates a linked focus task; completing it prompts the page update inline.
- Question: textbooks, pleasure reading, or both? (Chapters vs zero structure — don't build for both blindly.)

### Weekly Review
- Gaps: priorities go into a void (overwritten weekly, displayed nowhere else, last week's never shown during review); "Carried forward" includes tasks due later this week (Friday review scolds about Saturday's task); nothing ever invites her to review; habit window inherits UTC bug.
- Quick wins: show last week's priorities read-only with did-it/carry-it; fix carried-forward to `due_date < today`; Sunday/Monday dashboard whisper ("Ready for a 2-minute look back? →"); add a "wins" list — actual titles of completed tasks, not just counts.
- Bigger: priorities become real pinned objects visible on the dashboard all week, checkable, closing the loop next Sunday.
- Question: would she do the review WITH Andrew (body-doubling ritual)? If so optimize for two-people-one-screen.

### Settings
- Gaps: focus-limit knob is dead; export with **no import/restore** is false safety; no auto-backup (one corrupted SQLite file = everything, the real local-only risk); notifications column with no code; category icons not editable.
- Quick wins: wire focus limit; weekly auto-backup to app-data `/backups/` keep-last-8; JSON import behind a confirm.
- Bigger: nothing — settings should stay boring.
- Question: desktop notifications at all? Recommendation: mobile-only (Session 10 already plans one calm morning reminder); desktop stays silent.

### Pomodoro
- Gap: a sidebar item leading to a placeholder = a daily dead click in primary nav. Trust damage for zero benefit.
- Quick win: hide the nav item until it exists (5 min).
- Bigger (if built): not a generic pomodoro — a **"start on this" timer attached to a focus task**: pick task → "work on this for 15" → calm fullscreen timer → gentle "keep going / done / switch". Task INITIATION is the ADHD problem; the countdown is the excuse. Pairs with `time_estimate_min` for free. One-tap lengths (10/15/25/45), never track "failed" sessions.
- Question: does she use timers today? Yes → build it as its own session. No → delete the nav item, revisit in a month.

### Cross-cutting
- **No day-boundary behavior anywhere** — focus reset, snooze wake, recurrence spawn, date grouping all need one "new day" moment (app-open check suffices; ~50-line housekeeping function fixes the three worst betrayals in one place).
- UTC vs local inconsistent across the data layer — one deliberate pass.
- Completed work is invisible app-wide — a lightweight Done log matters as much as not losing tasks.
- Tags: full schema, zero code — recommend explicitly deferring (over-engineering trap).
- `due_time` renders if present but is unsettable — schema orphan.
- No keyboard shortcuts beyond in-modal Enter/Escape.
